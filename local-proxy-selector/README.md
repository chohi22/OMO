# Local Reverse Proxy Server

CLI로 설정된 로컬 포트를 외부에서 접속할 수 있도록 리버스 프록시하는 Node.js 서버입니다.

## 기능

- **CLI로 타겟 포트 설정**: `-t` 옵션으로 프록시할 포트 지정
- **외부 접속 지원**: `0.0.0.0`에서 리슨하여 외부에서 접속 가능
- **상태 모니터링**: `/status` 경로에서 현재 프록시 설정 확인
- **자동 프록시**: 루트 경로(`/`) 접속 시 타겟 서비스로 자동 연결

## 설치

```bash
npm install
```

## 사용법

### 기본 실행

```bash
# 3000번 포트로 프록시 설정
node server.js -t 3000

# 또는 직접 포트 번호 입력
node server.js 3000

# 환경변수 사용
PROXY_TARGET_PORT=3000 node server.js
```

### 도움말

```bash
node server.js --help
```

### 옵션

| 옵션 | 설명 |
|------|------|
| `-t, --target <port>` | 프록시할 타겟 포트 지정 (필수) |
| `-h, --help` | 도움말 출력 |

## 접속 방법

### 1. 외부에서 프록시 서비스 접속

```
http://[서버IP]:8888/
```

예시:
- `http://192.168.1.100:8888/` → `localhost:3000/`
- `http://192.168.1.100:8888/api/users` → `localhost:3000/api/users`

### 2. 상태 모니터링 페이지

```
http://[서버IP]:8888/status
```

현재 프록시 설정과 타겟 서비스 상태를 확인할 수 있습니다.

### 3. API 상태 확인

```
http://[서버IP]:8888/api/status
```

JSON 형식으로 상태 정보를 반환합니다:

```json
{
  "proxyPort": 8888,
  "targetPort": 3000,
  "targetUrl": "http://localhost:3000",
  "isTargetReachable": true,
  "configuredAt": "2024-01-01T00:00:00.000Z"
}
```

## 외부 접속 설정

### 같은 로컬 네트워크 (WiFi/LAN)

1. 서버 IP 확인:
```bash
# Linux/Mac
ip addr show | grep "inet " | head -1
# 또는
hostname -I
```

2. 클라이언트에서 해당 IP로 접속:
```
http://192.168.1.100:8888/
```

### 인터넷에서 접속 (포트 포워딩)

라우터/공유기 설정에서 포트 포워딩 필요:

```
외부 포트: 8888 → 내부 IP: [서버IP]:8888
```

### 터널링 서비스 사용

#### ngrok
```bash
ngrok http 8888
```

#### Cloudflare Tunnel
```bash
cloudflared tunnel --url http://localhost:8888
```

     │  http://[IP]:8888/     │                          │

## 라우팅 규칙

| 경로 | 동작 |
|------|------|
| `/` | 타겟 서비스로 프록시 |
| `/status` | 상태 모니터링 페이지 |
| `/api/status` | 상태 정보 JSON API |
| `/api/*` | API 엔드포인트 (예약) |
| `/*` | 모든 경로 타겟 서비스로 프록시 |

## 예시 사용 시나리오

### 1. 로컬 개발 서버 공유

```bash
# 로컬 React 개발 서버 (3000번)을 팀원과 공유
node server.js -t 3000

# 팀원은 http://[내IP]:8888 로 접속 가능
```

### 2. API 서버 외부 노출

```bash
# 로컬 API 서버 (5000번)을 외부 테스트용으로 노출
node server.js -t 5000
```

### 3. 모바일 기기 테스트

```bash
# 로컬 개발 서버를 같은 WiFi의 모바일 기기에서 테스트
node server.js -t 5173

# 모바일 브라우저에서 http://[PC_IP]:8888 접속
```

## 주의사항

- 타겟 포트는 반드시 지정해야 함 (`-t` 옵션 또는 `PROXY_TARGET_PORT` 환경변수)
- 외부 접속 시 방화벽에서 8888 포트를 개방해야 함
- 라우터에서 포트 포워딩 설정이 필요한 경우가 있음
- 프로덕션 환경에서는 적절한 보안 설정 필요
